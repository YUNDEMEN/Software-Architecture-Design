软件架构设计为什麽难？因为它是跨越现实世界（问题领域）到计算机领域（解决方案）之间的一座桥，需求分析是明确“问题领域”，将要解决的问题以
"功能 + 质量 + 约束"的形式定义下来，但需求不管做得再细，也没有打破“系统是黑盒子”这一点，软件架构设计就是完成从面向问题领域到面向解决方案的转换
（进入系统黑盒子）
